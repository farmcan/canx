# Naming and Positioning

**Chosen name:** `CanX`
**Repository name:** `canx`

## Why this name

- short and easy to remember
- broad enough to stay reusable beyond `Tradex`
- does not overcommit the implementation to only one underlying framework
- distinct enough from downstream business repositories

## Rejected directions

- `tradex-orchestrator`: too tied to one product
- `agent-runtime`: too generic and overloaded
- `openclaw-*`: misleading because the repository is not limited to OpenClaw
- `infinite-loop-*`: undesirable framing; the system should run bounded loops, not uncontrolled loops

## Positioning statement

`CanX` is a bounded multi-agent supervisor for Codex-centric software delivery.
